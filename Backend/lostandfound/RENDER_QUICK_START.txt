# FindEra Backend - Render Deployment Quick Reference

## 🎯 QUICK START DEPLOYMENT

### 1. Commit All Changes
git add .
git commit -m "feat: Prepare backend for Render deployment"
git push origin main

### 2. Render Web Service Configuration

Service Details:
- Name: findera-backend
- Environment: Node
- Build Command: npm install
- Start Command: npm start
- Root Directory: Backend/lostandfound
- Plan: Free (or Starter $7/month for production)

### 3. Environment Variables (Add These on Render)

NODE_ENV=production
PORT=5000
BASE_URL=https://findera-backend.onrender.com
DB_URL=mongodb+srv://username:password@cluster0.mongodb.net/lostandfound?retryWrites=true&w=majority
JWT_SECRET=generate-a-strong-random-string-here
JWT_EXPIRE=7d

### 4. Verify Deployment (After Service is Live)

# Check service is running
curl https://findera-backend.onrender.com/health

# Check API info
curl https://findera-backend.onrender.com/

# Open Swagger UI in browser
https://findera-backend.onrender.com/api-docs

### 5. Update Frontend

Update your frontend API base URL:
const API_BASE_URL = 'https://findera-backend.onrender.com'

### 6. MongoDB Setup

1. Go to MongoDB Atlas
2. Click Network Access → Add IP Address
3. Allow: 0.0.0.0/0 (all IPs)
4. Confirm username and password

## 🆘 TROUBLESHOOTING QUICK FIXES

### Service stuck in build?
- Check Render logs: render.com/dashboard → findera-backend → Logs
- Clear build cache and redeploy

### MongoDB connection error?
- Verify DB_URL is correct
- Check MongoDB IP whitelist is 0.0.0.0/0
- Test connection string with `mongosh`

### CORS errors?
- Add frontend URL to CORS origins in server.js
- Redeploy backend
- Clear browser cache (Ctrl+Shift+Delete)

### Swagger shows "undefined"?
- Set BASE_URL environment variable
- Wait 5 minutes for deployment to settle
- Restart service

### Free tier spinning down?
- Upgrade to Starter plan ($7/month)
- Or setup uptime monitoring ping every 14 minutes
